# Design Spec: Core Learning Loop Integration

> Date: 2026-04-16 | Feature: Deepen Core Learning Loop | Status: Approved

---

## 1. Overview

**Feature Summary:** Connect Quiz → AI Tutor → Flashcard creation into a seamless flow that helps students convert quiz mistakes into active learning.

**Business Value:**
- Increase flashcard creation from wrong answers by 40%
- Provide immediate AI explanations for mistakes
- Show unified "Knowledge Health Score" combining quiz and flashcard metrics

**Success Metrics:**
- Quiz → AI explanation: 30% conversion
- AI → Flashcard creation: 50% conversion  
- Knowledge Health Score: 60% student adoption

---

## 2. User Flow

```
Quiz Complete → Wrong Answers Shown (max 5)
        ↓
Tap "Get AI Explanation" 
        ↓
AI Tutor opens with question pre-loaded
        ↓
AI explains → Shows "Save as Flashcard" button
        ↓
Tap → Flashcard created + added to review queue
        ↓
Dashboard shows Knowledge Health Score
```

---

## 3. Technical Architecture

### 3.1 Data Model

**New Table: `learningLoopEvents`**
```sql
learningLoopEvents {
  id: uuid
  userId: uuid
  quizId: string
  questionId: string
  aiSessionId: uuid (nullable)
  flashcardId: uuid (nullable)
  triggeredBy: 'quiz' | 'tutor' | 'learning-loop'
  outcome: 'explained' | 'flashcard_created' | 'skipped'
  createdAt: timestamp
}
```

**Schema Changes:**
- `quizResults`: add `wrongAnswersProcessed` boolean (default: false)
- `flashcardDecks`: add `autoGeneratedFrom` string (nullable)
- `aiChatSessions`: add `triggeredBy` enum

### 3.2 API Changes

| Endpoint | Method | Purpose |
|----------|--------|---------|
| `/api/learning-loop/convert-wrong-answers` | POST | Trigger AI explanations for wrong answers |
| `/api/learning-loop/generate-flashcards` | POST | Create flashcards from AI explanations |
| `/api/progress/knowledge-health` | GET | Combined score from quiz + flashcards |

### 3.3 Frontend Components

| Component | Path | Purpose |
|-----------|------|---------|
| WrongAnswerExplanationButton | `src/components/Quiz/` | "Get AI Explanation" on wrong answers |
| AIExplanationModal | `src/components/AI/` | Pre-loaded question + "Save as Flashcard" |
| KnowledgeHealthWidget | `src/components/Dashboard/` | Combined score display |
| LearningLoopToast | `src/components/Feedback/` | "X flashcards added to queue" |

## 4. Auto-Convert Logic

**Default: Enabled** (can be disabled in settings)

```typescript
// quiz-complete flow
if (autoConvertEnabled && wrongAnswers.length > 0) {
  const topWrong = wrongAnswers.slice(0, 5);
  
  for (const wrong of topWrong) {
    // 1. Log learning loop event
    await logLearningLoopEvent({
      quizId,
      questionId: wrong.questionId,
      triggeredBy: 'learning-loop'
    });
    
    // 2. Auto-create flashcard
    await createFlashcardFromQuestion(wrong);
  }
  
  // 3. Show toast
  toast.success(`${topWrong.length} flashcards added to your review queue`);
}
```

## 5. Knowledge Health Score

**Formula:**
```
KnowledgeHealth = (QuizAccuracy * 0.6) + (FlashcardRetention * 0.4)

Where:
- QuizAccuracy = (correct answers / total questions) from last 30 days
- FlashcardRetention = percentage of cards rated "Easy" or "Good" in SM-2
```

**Display:**
- Dashboard: Circular progress ring (0-100%)
- Progress page: Detailed breakdown by subject

## 7. Edge Cases

| Scenario | Handling |
|----------|----------|
| No wrong answers | Show success message, no AI button |
| >5 wrong answers | Cap at 5, show "See all in Review Mode" |
| AI quota exceeded | Show "Try again later" with retry option |
| Offline during quiz | Queue auto-convert for sync when online |
| Flashcard already exists | Skip duplicate, show "Already in queue" |

---

## 8. Backward Compatibility

- `wrongAnswersProcessed` defaults to `false` for existing results
- `autoGeneratedFrom` nullable for existing decks
- AI sessions without `triggeredBy` default to `'tutor'`
- Feature flag for gradual rollout
